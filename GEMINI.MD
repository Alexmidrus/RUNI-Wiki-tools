# 📜 Глобальные правила проекта и Контекст для ИИ

Этот документ описывает архитектуру, стандарты кода и правила работы над проектом. Он также служит системным промптом для искусственного интеллекта (Gemini / Claude / и др.).

## 1. О проекте
* **Название:** RUNI Wiki Tools
* **Главная задача:** CLI-инструментарий для офлайн-редактирования контента MediaWiki (статьи, шаблоны, категории, глобальные скрипты/стили) с возможностью двусторонней синхронизации (pull/push) через MediaWiki API.
* **Целевая вики по умолчанию:** `https://solarmeta.evecraft.ru`
* **Язык:** Python 3.13+

## 2. Стек технологий
* **Основной фреймворк:** Чистый Python (CLI-приложение, единая точка входа `main.py`).
* **Сетевые запросы:** Модуль `requests`.
* **База данных:** Без БД. Используется файловая система, все скачиваемые/генерируемые файлы находятся внутри папки `data/` (в форматах YAML/JSON и сырые файлы `.wiki`, `.css`, `.js`).
* **Управление зависимостями:** Виртуальное окружение `.venv`, `pip` + `requirements.txt`.
* **Доп. инструменты:** Конфигурация через `.env` (переменные окружения).

## 3. Стандарты кода (Strict Rules)
* **Линтинг и форматирование:** Использование `Ruff`. Максимальная длина строки — 100 символов.
* **Типизация:** Строгая типизация обязательна. Все функции, методы и классы должны иметь Type Hints. Для сложных структур используем модуль `typing` (включая `Optional`, `Sequence`, `Dict` и т.д.).
* **Документация:** Каждая публичная функция, класс и модуль должны сопровождаться Docstrings в стиле **Google**.
* **Обработка ошибок:** Запрещено использовать пустой `except:`. Исключения должны быть конкретными. Для бизнес-логики и критичных ошибок используем `raise RuntimeError("Подробное описание")` по аналогии с имеющимся кодом.
* **Именование:**
  - Переменные и функции: `snake_case`
  - Классы: `PascalCase`
  - Константы: `UPPER_SNAKE_CASE`

## 4. Базовая структура проекта
Код модульный и разделен по смыслу:
* `script/` — модули приложения, реализующие логику работы (API вики, UI в консоли, логика импорта/экспорта различного контента, парсинг URL).
* `data/` — единственное место для сохранения результатов локально (статьи, категории, шаблоны, URL-списки, глобальные стили/скрипты). **ВАЖНО**: скрипты не должны писать файлы за пределы папки `data/`.
* `roles/` — промпты и инструкции для специализированных агентов ИИ.
* `main.py` — единый CLI entrypoint (поддерживает команды типа `template`, `article`, `category`, `urls`, `push`, `push-templates`).
* `.env` — файл настроек окружения (хранение секретов API: логины, пароли бота, URL). Не коммитится в репозиторий. Совместим с `os.getenv()`.

## 5. 🤖 Инструкция для ИИ (Системный Промпт)
Привет, Gemini! При генерации любого кода для этого проекта ты должен выступать в роли Senior Python Developer и неукоснительно соблюдать правила из этого документа, а также правила из `README.md`. 
Твой код должен быть:
1. Чистым, читаемым и совместимым с архитектурой CLI-приложения.
2. Строго типизированным (`Optional`, `Dict`, `List`, `Any` и т.д.).
3. Покрытым базовыми комментариями (Google Docstrings).
4. Безопасным (не хранить пароли в коде, использовать переменные окружения через загрузчик `.env` файлов).
5. Готовым к production и безопасным (например, обработка SSL при ключах `--insecure`).
6. **Все выходные данные должны сохраняться строго внутри директории `data/`.**
